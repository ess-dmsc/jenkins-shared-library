        • PipelineBuilder(java.lang.Object script, java.lang.Object containerBuildNodes, java.lang.Object hostMounts = "")

          The branch name is used to generate the Docker container name; some percent-encoded characters, like '/' and '#', are replaced with '_'.
          throws:
          IllegalArgumentException if build node keys have wrong type or JOB_NAME has wrong number of components.
          Parameters:
          script - reference to the current pipeline script (this in a Jenkinsfile)
          containerBuildNodes - map with string keys and ContainerBuildNode values
          hostMounts - string with host directories to be mounted read-only in container, with the form "src1:dst1,src2:dst2,..."

        • java.lang.Object createBuilders(groovy.lang.Closure pipeline)

          Create a map of builders to be passed to a Jenkins parallel step. Standard Jenkins pipeline steps are executed on the allocated build node outside the container. To run commands with the container, call the Container methods on the closure parameter.

          Parameters:
          pipeline - parameterised closure defined with curly braces and the parameter name before an arrow, where the parameter uses the Container interface
          Returns:
          Map of string keys and builder values

        • java.lang.Object handleFailureMessages()

          Handle failures that occurred inside stage blocks. If failures occurred, the associated failure messages are sent using the activated notification channels.

        • java.lang.Object stage(java.lang.String name, groovy.lang.Closure stageCommands)

          Get a Jenkins pipeline stage with automated failure messages. If an exception occurs inside the stage block, the messages are saved for handleFailureMessages(), identifying the stage name.

          throws:
          Exception if any command in the block throws an exception
          Parameters:
          name - stage name
          stageCommands - block of commands between curly braces
